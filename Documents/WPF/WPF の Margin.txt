WPF の Margin

// 基本
ウィンドウの外周部に10ピクセル
コントロール間に5ピクセル
コントロールの集まりと集まりの間に10ピクセル
グリッドの境界線をまたぐなら左に2ピクセル、右に3ピクセル
// 「兄さん」と覚える

// 集まりかどうか
グリッドの境界線をまたぐにおいて「兄さん」にするか5/5にするか迷うときがある
見た目で主観的に判断するとブレる
データや機能としての論理的関連性があるかどうかで判断する
たとえば左に文書リスト、右に本文を表示するなら、これらの関連性は、5px に近づけるほど強くない
しかし、左に「追加」ボタンなど、右に「保存」ボタンなどを置くなら、これらは近づけるべき

// 「兄さん」の問題
たとえばウィンドウを左右方向に均等に3分割してそれぞれに TextBox を置いても幅が一致しない
たいてい問題ないが、1ピクセルも違ってはいけない場合には他の配置方法を考える
