niceWriter

自動保存されるためパソコンが落ちても本文を失わないエディターで、文字数がカウントされ、プロンプトが設定ファイルに入っているためボタン一つで文章をチェックできるように。

メールへの返信を考え、原文を貼り付けておけば全てのトピックに言及できたか調べることができたり、返信を自動生成したりも。

また、循環翻訳も行えて、原文、訳文、原文の言語に戻したものの三つを含めて保存できれば、中国とのやりとりに役立つ。その場合のことも考えて、0個以上のタグを選べるようにするか。あまりないが、一つのメッセージを複数の取引先に送って見積もりを出してもらうこともある。

そこにファイルや写真を添付できれば Instagram などにも使えそうだが、ややこしくなりそうなので迷うところ。

というのも、ファイル管理においては、同じセッションの写真においても、Instagram と Facebook とでは載せられる数が異なるので前者を絞り込むことになるし、写真ごとのコメントをつけた方がコンテンツとして面白く、自分のサイトに載せるにおいては、ログインなしで見られるものとログインが必要なものを区別できた方がセキュリティーやプライバシーに貢献する。

そういったところを UI に入れると、たとえタブで区切るなどしても複雑になる。

それよりは、別のアプリをつくり、それが画像ビューアーのような UI を持ち、並び替えができて、それぞれにコメントができて、ファイル名で並び替えてのアップロードに向くように「Instagram に載せるものだけをデスクトップへ連番でコピー」などができて、そこに雰囲気やトーンを検証するツールの JSON ファイルも流し込めば本文だと自動認識される方がシンプル。文章を入れ忘れたら保存できないようにすれば、入れ忘れにより関連性が喪失することも起こりえない。

そういうツールだと短い動画も入れられる。さすがに大きなものはストレージを圧迫するが、数十 MB 程度の面白動画なら写真と同じリストに入れて並び替えまでやってよい。
