OneTeam

プロジェクト管理を含むグループウェアと顧客管理をできるだけシンプルに。

レイヤーごとの仕様で書いてみる。つまり、最終的にどういうシステムにしたいか最初に決めたうえ、最初はミニツールくらいのものにする。

「商品データベースや顧客管理までいっそのこと」と思っていたが、さすがに無理やり。OAuth か何かの利用により、どこかでログインしたらどのサイトも開けるように。そのうえで、それぞれを他社にも魅力的なシステムとして構成。

顧客管理はタスク管理やファイル管理と連動してよいのでグループウェアに入れてもよさそうに思えるが、その理屈でいくと、顧客情報はサイトのアクセスログとも連動させたく、やっぱり一つに……ということになる。

しかし、そうすると、もう何もかもを一つにまとめたくなる。それをやるとセキュリティーのリスクが増大するし、スケーラビリティーも下がる。

システムを常識的に分割し、間を API でつなぐ。UI が似ていれば、別ウィンドウを開くリンクがちょくちょくあっても仕事に大きな影響はない。

このシステムに持たせたい機能は、だいたい次の通り。

* 翻訳
* ユーザー管理
* マルチスレッドのフォーラム（チケットっぽく、終わったら閉じる）
* 静的ファイル（フォーラムでは流れ、こちらでは階層的に。カタログなどに）
* ウィキ（ここにもタスクを関連づけられる）
* メール送信
* ダッシュボード
* 外部公開 URL
* 操作ログ

ほかにも探したうえ、なくてよいものを削り、できるだけシンプルに。

ウィキを翻訳しきれない問題がある。

一方、競合分析などにおいてはウィキは非常に重要。

フロー型とスタック型の違いが大きいので、翻訳できないスタック型を別につくるのも選択肢。そのほうがすっきりするか。
