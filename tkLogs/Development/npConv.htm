npConv

古い写真をニコルが見たがっている。

ニコルが生まれる直前に nikoP というウェブシステムを書いた。当時は VPS を持っていて、容量もあったので、そこへの写真のアップロードにより、整理、家族・親戚との共有、バックアップを兼ねていた。

類似サービスがたくさんあるが、全ての原版を残したく、また、二重管理（ローカルで整理し、そのうち見せたいものだけを類似サービスに載せるなど）を避けたかった。

VPS なので RDP で接続可能で、たまにつないでは、サーバー側で自分自身の Subversion レポジトリーに写真をコミットし、それをローカルのパソコンにレポジトリー経由でダウンロードしていた。

この方法は、二重管理にならず、いつサーバーあるいは自宅のパソコンが壊れても絶対に写真を失わないので完璧だ。

しかし、そういうことができる容量を持つ VPS は月額1万円を軽く超える。さすがにもったいなかったので、それがたまたま Windows Server 2012 で、OS のサポート期限が迫っていることでサービス終了を言ってきたときに、容量が大きくてレポジトリーを作り放題のサーバーに切り替えた。そちらも月額5千円くらいかかっているが、コストは2/5くらいになったと思う。

サーバーの切り替え時に家族の LINE グループを作り、日々の写真をそちらに載せるようになった。その方がリアルタイム性があるし、幼少期は短いので、それはそれでアリだと思っている。

それとは別に、「サーバーの nikoP にアップロードし、サーバーでコミットし、ローカルにダウンロードした、ディレクトリー構造などが特殊だからローカルでの閲覧に適さない膨大な古い写真」をそろそろ整理し、コスト2/5の Subversion レポジトリーに入れ直したい。

その際には、文字として残さないと失われる育児メモくらいは引き継ぎたい。

これは完全に個人的なプロジェクトなのでレポジトリーを公開しても仕方ないが、隠す理由もないので……。
