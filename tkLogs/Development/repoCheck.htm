OK コミットの件はやめておく。仕様を大きく変更したなどで「たまに最近の差分をザッと」というチェックは確かに有用。しかし、最後が OK コミットでないと警告が表示されるようにしては、「スペルミスを直したが、アーカイブまでは更新しない」のような状態でもずっと警告が出る。

✓ GitKraken や VSC との数の一致を確認 ∞

ステージされていなければ untracked と modified が区別され、ステージされていれば両方が staged とされるか。

untracked と modified がいずれも複数含まれるレポジトリーにおいて、ちょびちょび（死語？）ステージしてみる。

GitKraken では、変更されたファイルの内容のうち半分だけステージされているときにリストでは二つのファイルに変更があるように表示される。repoCheck では 1 changed で、その方が自然。

その点以外は一致。
